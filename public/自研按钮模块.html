自研按钮模块

今天我们学习一个能让作品更聪明的小模块。目标不是背知识，而是做出能看见、会判断、能行动的小作品。

生活里在哪里见过？

电梯按钮、门铃、游戏手柄都在用按钮。今天我们要让掌控板听懂「我按下了」这个动作。

本课目标

  • 能说出 自研按钮模块 在生活中的用途
  • 能完成基本接线或软件准备
  • 能用 Mind+ 积木读取或控制它
  • 能用「如果……那么……」做一个小判断

它从哪里来？

很久以前，人们就用开关控制灯和机器。按钮是最容易理解的输入：按下去，机器就知道有人发出了命令。

它能做什么？

按钮就像给掌控板的「举手信号」。按下时，程序可以开始一件事。

准备与接线

  • V 接掌控板 3V3 或 5V（看模块标识）
  • G 接掌控板 GND
  • S 接掌控板 P 口，例如 P0、P1、P2、P8
  • Mind+ 中读取对应 P 口的数字值
  • 按下通常读到 1，松开通常读到 0（以实际模块为准）

注意：这里不要写成“输入口”就结束，要明确告诉学生：按钮的 S 信号线接到掌控板的 P 口。

按钮接线图

先接 V 和 G，再把 S 接到一个 P 口。程序里读取哪个 P 口，线就要接到哪个 P 口。

工作原理

按钮内部像一座小桥。按下时电路接通，掌控板从 P 口读到一个数字信号；松开时电路断开，P 口读到另一个数字信号。我们用这个 0/1 变化来判断“有没有按下”。

按键消抖

机械按键在刚按下或刚松开时，金属触点可能会轻轻弹跳。掌控板读得很快，就可能把一次按下误认为很多次。

  • 检测到按下后，先等待 20~50 毫秒
  • 再读取一次 P 口，确认仍然是按下
  • 等按钮松开后，再允许下一次触发
  • 抢答器、计数器、切换灯光都需要消抖

三个学习任务

任务 1

按一下按钮，让屏幕显示「你好」。

步骤：先完成接线或软件准备，再用 Mind+ 读取/控制模块，最后加入条件判断并观察结果。

接线答案
按钮 V 接掌控板 3V3 或 5V（按模块标识）
按钮 G 接掌控板 GND
按钮 S 接掌控板 P0
Mind+可视化程序答案
当掌控板启动
重复执行
如果 数字读取 P0 = 1
等待 0.03 秒（按键消抖）
如果 数字读取 P0 仍然 = 1
清空屏幕
在屏幕第 1 行显示「你好」
等待 直到 数字读取 P0 = 0（防止一直重复触发）
任务 2

按按钮切换RGB灯颜色。

步骤：先完成接线或软件准备，再用 Mind+ 读取/控制模块，最后加入条件判断并观察结果。

接线答案
按钮 V 接掌控板 3V3 或 5V（按模块标识）
按钮 G 接掌控板 GND
按钮 S 接掌控板 P0
RGB灯使用掌控板板载RGB灯，不需要额外接线
Mind+可视化程序答案
当掌控板启动
设置 变量「颜色编号」= 0
重复执行
如果 数字读取 P0 = 1
等待 0.03 秒（按键消抖）
如果 数字读取 P0 仍然 = 1
将「颜色编号」增加 1
如果 颜色编号 = 1，那么 RGB灯设为红色
如果 颜色编号 = 2，那么 RGB灯设为绿色
如果 颜色编号 = 3，那么 RGB灯设为蓝色，并把颜色编号改回 0
等待 直到 数字读取 P0 = 0（松手后才允许下一次切换）
任务 3

做一个两人抢答器。

步骤：先完成接线或软件准备，再用 Mind+ 读取/控制模块，最后加入条件判断并观察结果。

接线答案
1号按钮：V 接 3V3/5V，G 接 GND，S 接 P0
2号按钮：V 接 3V3/5V，G 接 GND，S 接 P1
两只按钮的 GND 必须都和掌控板 GND 相连
用掌控板屏幕显示抢答结果，不需要额外显示屏
Mind+可视化程序答案
当掌控板启动
设置 变量「获胜者」= 0
重复执行
如果 P0 = 1 且 获胜者 = 0
等待 0.03 秒（1号按钮消抖）
如果 P0 仍然 = 1，那么 设置 获胜者 = 1，屏幕显示「1号抢答成功」
如果 P1 = 1 且 获胜者 = 0
等待 0.03 秒（2号按钮消抖）
如果 P1 仍然 = 1，那么 设置 获胜者 = 2，屏幕显示「2号抢答成功」
如果 掌控板 B键 被按下，那么 获胜者 = 0，屏幕显示「准备抢答」

本课总结

  • 我知道 自研按钮模块 的作用和生活场景
  • 我能检查 VCC、GND、信号线是否接对
  • 我能在 Mind+ 中找到需要的主控板和扩展
  • 我能把读数、判断、输出连成一个完整作品

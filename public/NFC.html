NFC

今天我们学习一个能让作品更聪明的小模块。目标不是背知识，而是做出能看见、会判断、能行动的小作品。

生活里在哪里见过？

刷公交卡、门禁卡时，机器会读到卡片身份。我们也能做一个刷卡作品。

本课目标

  • 能说出 NFC 在生活中的用途
  • 能完成基本接线或软件准备
  • 能用 Mind+ 积木读取或控制它
  • 能用「如果……那么……」做一个小判断

它从哪里来？

公交卡、门禁卡常用RFID或NFC技术。它们让卡片不用插进去，也能被机器识别。

它能做什么？

它可以读取卡片编号，常用于刷卡签到、门禁和身份识别。

准备与接线

  • 先把模块左上角拨码开关拨到 IIC / I2C 模式
  • SCL 接掌控板 SCL
  • SDA 接掌控板 SDA
  • VCC 接掌控板 3V3
  • GND 接掌控板 GND
  • 右侧 SCK/MISO/MOSI/SS 是 SPI 引脚，本课不用接

这块 NFC 模块同时引出了 IIC 和 SPI。我们这节课走 IIC 协议，所以只接上方 SCL、SDA、VCC、GND 四个接口，并检查拨码开关是否在 IIC 模式。

IIC接线图

这张图把模块、掌控板、卡片和 IIC 四根线放在一起，帮助学生看清连接关系。

工作原理

NFC 模块靠近卡片后，会读取卡片里的 UID 编号。掌控板通过 IIC 协议和模块通信：SCL 像时钟节拍，SDA 负责传数据。程序拿到 UID 后，就能判断这张卡是不是允许通过。

IIC拨码开关

左上角拨码开关用来选择通信方式。使用本课程序前，要把拨码设置到 IIC/I2C 模式；如果拨到 SPI，SCL/SDA 程序就读不到卡片。

  • 本课只接 SCL、SDA、VCC、GND
  • SCK、MISO、MOSI、SS 是 SPI 接口，本课不使用
  • 先读出 UID，再把 UID 填入判断程序
  • 卡片要贴近白色天线区域再读取

三个学习任务

任务 1

读取一张卡片编号。

把卡片靠近模块，读取并显示卡号。

接线答案
拨码开关拨到 IIC / I2C 模式
SCL 接掌控板 SCL
SDA 接掌控板 SDA
VCC 接掌控板 3V3
GND 接掌控板 GND
Mind+可视化程序答案
当掌控板启动
初始化 NFC 模块（IIC）
重复执行
如果 NFC 读取到卡片
设置 变量「卡号」= 读取到的 UID
屏幕显示「卡号」和变量「卡号」
等待 0.5 秒，避免同一张卡连续刷屏
任务 2

判断是不是指定卡片。

把一张卡设为通行卡，其他卡不通过。

接线答案
拨码开关拨到 IIC / I2C 模式
SCL/SDA/VCC/GND 分别接掌控板 SCL/SDA/3V3/GND
先用任务1读出允许卡的 UID，并记录下来
Mind+可视化程序答案
当掌控板启动
初始化 NFC 模块（IIC）
设置 变量「允许卡号」= 课堂记录的 UID
重复执行
如果 NFC 读取到卡片，设置 变量「卡号」= UID
如果 卡号 = 允许卡号
那么 RGB灯绿色，屏幕显示「通过」
否则 RGB灯红色，屏幕显示「未授权」
任务 3

做一个刷卡签到器。

刷到指定卡后显示签到成功，并避免重复刷太快。

接线答案
拨码开关确认在 IIC / I2C 模式
只接上方 IIC 四线：SCL、SDA、VCC、GND
卡片靠近模块白色天线区域读取
板载屏幕显示签到结果，不需要额外显示屏
Mind+可视化程序答案
当掌控板启动
初始化 NFC 模块（IIC）
设置 变量「已签到」= 0
重复执行
如果 读取到指定 UID 且 已签到 = 0
那么 屏幕显示「签到成功」，RGB灯绿色，已签到 = 1
等待 2 秒，防止同一张卡连续触发
如果按下 B键，则 已签到 = 0，屏幕显示「准备签到」

本课总结

  • 我知道 NFC 的作用和生活场景
  • 我能检查 VCC、GND、信号线是否接对
  • 我能在 Mind+ 中找到需要的主控板和扩展
  • 我能把读数、判断、输出连成一个完整作品
